# GYM Dashboard Internal MVP Flow

## 역할 분리

- Vercel: 대시보드 화면 배포
- Supabase: 매장, 직원, 경쟁사 리포트, 주간 업무, 매출 분석 결과 저장
- Codex: 주 1회 경쟁사 채널 조사와 리포트 작성
- Gemini API: 업로드된 Codex 리포트를 직원별 실행 업무로 변환

## 주간 운영 흐름

1. 대표 또는 운영자가 Codex로 경쟁사 분석을 수행한다.
2. 분석 결과를 `주간 경쟁사 리포트` 화면에 붙여넣거나 파일로 업로드한다.
3. 리포트는 Supabase `competitor_reports`에 저장된다.
4. `AI 업무 생성` 버튼을 누르면 Gemini API가 아래 데이터를 함께 읽는다.
   - 이번 주 경쟁사 리포트
   - 직원 이름, 직무, 성향, 근무 가능 시간, 강점
   - 지난주 미완료 업무와 완료 메모
5. Gemini API는 직원별 업무를 생성한다.
6. 생성된 업무는 Supabase `weekly_tasks`에 저장된다.
7. 직원은 모바일 또는 PC에서 담당 업무를 체크한다.
8. 대표는 주간 리뷰 화면에서 완료율, 미완료 사유, 다음 대응 전략을 확인한다.

## 경쟁사 리포트 입력 항목

- 조사 주차
- 경쟁사별 네이버 플레이스 변화
- 경쟁사별 블로그 신규 키워드
- 경쟁사별 인스타그램 콘텐츠 흐름
- 가격, 시설, 콘텐츠, 리뷰, 검색 노출 위협
- ITN이 이번 주 바로 대응해야 할 액션

## Gemini 업무 생성 기준

- 관계형 직원: 재등록 상담, 후기 요청, 장기회원 케어
- 세일즈형 직원: 신규 상담, 체험권 전환, 가격 민감 고객 설득
- 분석형 직원: 매출/미수금/회원 변동 점검
- 콘텐츠형 직원: 블로그, 릴스, 플레이스 소식, 전후 사례 정리
- 운영관리형 직원: 체크리스트, 일정, 완료 여부, 누락 업무 관리

## 내부 MVP 우선순위

1. 직원 정보 Supabase 저장
2. 주간 경쟁사 리포트 저장
3. Gemini API 업무 생성
4. 직원별 업무 체크
5. 대표 주간 리뷰
6. 매출 엑셀 업로드와 분석 결과 연결
